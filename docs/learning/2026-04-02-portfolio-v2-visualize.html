4. 修正前後の比較

カードの存在感の出し方

2020年代前半: 枠線を目立たせる → 2025-26年: 背景色の差 + shadow で浮かせる

Before ❌
border: 1px solid rgba(gold, 0.25); /* 枠線が目立つ = 古い印象 */
After ✅
border: 1px solid rgba(gold, 0.1); background: rgba(15,23,42, 0.5); box-shadow: 0 8px 40px rgba(0,0,0, 0.6);

テキストコントラスト

雰囲気重視で透明度を下げすぎると読めない → WCAG 4.5:1 を意識してバランス調整

Before ❌
--ink2: rgba(cream, 0.5); /* 読みづらい */ --ink3: rgba(cream, 0.2); /* ほぼ見えない */
After ✅
--ink2: rgba(cream, 0.7); /* しっかり読める */ --ink3: rgba(cream, 0.35); /* 存在が分かる */

7. 初心者向けチェックリスト

  • "use client" は useState / useEffect / ブラウザAPI を使うコンポーネントにだけ付ける
  • Next.js 16 では params は Promise → 必ず await params する
  • Tailwind CSS v4 では tailwind.config.js は不要 → @theme inline で設定
  • ダークテーマの半透明テキストは実際にブラウザで読みやすさを確認する
  • 背景装飾は position: fixed + -z-10 でコンテンツの後ろに配置
  • iframe の pointer-events: none を忘れるとプレビューがクリックを奪う
  • デザイントークンは「色名」ではなく「役割名」で定義（--accent > --orange）
  • カードの存在感は枠線ではなく背景色の差 + shadow で出す（2025-26年トレンド）
